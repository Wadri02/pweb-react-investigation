# Chakra UI v3

## What is it?
Chakra UI is an accessible React component library created by Seun Adebayo and released in 2019. Its philosophy is to provide components that are accessible by default (WAI-ARIA), composable, and highly customizable through a token system and style props.

Version 3 (2024) was a major rewrite. The most visible change: `ChakraProvider` was replaced by `Provider` from `@chakra-ui/react`, and the theming system migrated entirely to CSS custom properties (CSS variables). Internally, Chakra v3 abandoned Emotion as its CSS-in-JS engine and adopted Panda CSS for better performance and Server Component compatibility.

Chakra v3 also introduced `colorPalette` as a standard prop on all components, simplifying the application of thematic colors, and the recipe system (`defineRecipe`) for component variants.

## What is it used for?
Building accessible and consistent user interfaces without writing CSS, with production-ready components that include keyboard handling, focus management, and correct ARIA roles.

In the real world: forms, modals, tooltips, drawers, and tables with correct accessibility without having to implement it manually.

## Differences between v2 and v3

| Feature | Chakra UI v2 | Chakra UI v3 |
|---------|-------------|-------------|
| Main provider | `ChakraProvider` | `Provider` (from `@chakra-ui/react`) |
| CSS engine | Emotion | Panda CSS |
| CSS variables | Generated by JS | Native CSS custom properties |
| Theming | `extendTheme()` | `createSystem()` + tokens |
| Colors | `colorScheme` prop | `colorPalette` prop |
| Server Components | Incompatible | Compatible |

## Key Concepts

**Token system** — Chakra defines a spacing scale, typography, colors, and shadows as tokens. Referenced with strings: `p="4"` (padding of token 4), `color="blue.500"`.

**colorPalette** — Prop in v3 that applies a semantic color palette to a component. Replaces `colorScheme` from v2. Example: `<Button colorPalette="blue">`.

**sx prop** — Escape hatch for arbitrary CSS. Accepts any CSS property plus Chakra tokens.

**Composition with `as`** — Many components accept `as` to change the underlying HTML element while keeping the styles.

**Recipes (defineRecipe)** — v3 system for defining component variants with a typed API, similar to Tailwind CVA.

## When to use it?
- When accessibility is a priority and you don't want to implement it manually.
- Projects that need a consistent design system with flexible theming.
- Teams that prefer the style props API to Tailwind classes.
- Dashboards, internal apps, admin panels.

## When NOT to use it?
- When you need full CSS control (Tailwind or CSS Modules).
- If the design is highly custom and Chakra components are a hard starting point.
- Projects with very restrictive bundle size (Chakra adds weight).
- When the design specifically follows Material Design (use MUI).

## Is it worth learning?
Chakra UI has good market demand, especially in startups and medium-sized companies. The learning curve is low: the props API is intuitive and the documentation is excellent. The v2 to v3 migration has considerable friction, so existing v2 projects may stay there. For new projects, v3 is the correct version.

## Alternatives

| Technology | When to choose it |
|------------|------------------|
| **Chakra UI v3** (this) | Accessibility, theming, composability, props API |
| **Material UI (MUI)** | Material Design, more components, more enterprise-focused |
| **Tailwind CSS** | Full design control, no pre-built components |
| **shadcn/ui** | Components copied into the project, Tailwind, maximum control |
| **Radix UI** | Accessible primitives without styles, build on top |

## Chakra UI or MUI?
Chakra is more flexible and easier to customize outside Material's design system. MUI has more components (DatePicker, DataGrid) and is more mature in enterprise settings. If the design must follow Material Design, MUI wins clearly. If you want accessibility with visual freedom, Chakra has less friction. For internal dashboards where appearance is secondary, MUI has more ready-made components. For consumer products with a custom design, Chakra is easier to adapt.

## What does the example in this branch do?
`src/App.tsx` configures Chakra v3's `Provider` and uses components like `Button`, `Box`, `Stack`, `Input`, `Text` with the token system and `colorPalette`. It demonstrates theming and how styles are applied with props directly on the components.
